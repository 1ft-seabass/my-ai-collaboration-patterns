# リファクタリング（治療）

指定した箇所をリファクタリングしてください。

## やること

- 指定範囲のコードを改善する
- 変更理由を説明する
- 既存の挙動を維持する

## 観点

- 重複の統合
- 長い関数の分割
- 命名の改善
- 不要コードの削除
- 責務が混ざりすぎていないか

## ルール

- **機能は変更しない**（挙動を維持）
- **指定範囲外に影響を与えない**
- **テストがある場合は必ず通す**

## やらないこと

- 機能追加・変更
- 指定範囲外への影響
- 大規模な設計変更（まず dev_review.md で相談）

## 推奨フロー

1. 事前にテストを作成（dev_testing.md）
2. リファクタリング実行
3. テストが通ることを確認
4. 変更内容を notes に記録
