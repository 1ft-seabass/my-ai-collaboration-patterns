ではここまで進捗をコミットしましょう。段階的に開発や知見がたまったと思うので、段階的に状況をコミットしてください。

## 📋 この指示書の理解チェック

作業を開始する前に、以下の内容を理解したことをチェックボックスで提示してください：

- [ ] git status と git diff で変更内容を確認する
- [ ] 機密情報が含まれていないかチェックする
- [ ] git log で従来のコミットスタイルを把握する
- [ ] ユーザーに AI 署名・プレフィックス・言語を確認する
- [ ] 段階的なコミット計画を提示し、承認を得る
- [ ] プッシュはせず、コミットのみで完了する

理解できましたか？ゴーサインをください。

---

## 絶対守るべきこと

- **プッシュはせずコミットのみを行う**（人間がレビュー後にプッシュ）
- **ノート・申し送り・タスクなどコミットする情報は API キーや機密情報を必ずプレースホルダーで扱う**
- **.gitignore で保護されている情報は絶対に漏洩させない**
- **コミット前に git diff を実行して変更内容を確認する**

## 実行手順

1. 変更内容の確認
   - `git status` で変更ファイルを確認
   - `git diff` で変更内容を確認
   - 機密情報（APIキー、パスワード、トークンなど）が含まれていないかチェック

2. 従来のコミットログを参考にスタイルを把握
   - `git log --oneline -10` で最近のコミットを確認
   - コミットメッセージの形式（feat:/fix:/docs: など）
   - 日本語 or 英語
   - AI 署名の有無（`Co-Authored-By: Claude` があるか）

3. ユーザーにコミットスタイルを確認
   - このリポジトリは AI 署名を入れますか？ [y/N]
   - コミットメッセージのプレフィックスは？（feat/fix/docs/refactor など）
   - 日本語 or 英語？

4. 開発コミット計画の提案
   - 開発経緯がわかるよう、段階的コミット計画を提示
   - ユーザーの承認を得る

5. 開発内容のコミット
   - 承認された計画に従って段階的にコミット
   - 確認したスタイルに従ってコミットメッセージを作成
   - コミット前に必ず git diff で内容を確認

6. コミット完了の報告
   - コミットした内容を簡潔に報告
   - プッシュはせず、コミットのみで完了
