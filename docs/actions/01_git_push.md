このセッションでプッシュ前の最終チェックを実施します。

## 対象範囲

このプロジェクトの実運用フローでは、プッシュ前チェックの時点で対象コミットは既にコミット済み・ステージ無し（クリーンな作業ツリー）になっているため、`git diff --cached` は常に空になります。**未pushコミットの差分**を対象にします。

## チェック項目

1. ドキュメント類の機密情報チェック（最重要）
   - 未pushコミットの全差分を対象（フォルダ・ブランチ構成に関わらず全ファイル）
     * upstream設定済み: `git diff @{u}...HEAD`
     * upstream未設定（初回push等）: `git diff <デフォルトブランチ>...HEAD`（例: `git diff origin/main...HEAD`）。それも取得できない場合は `git log --oneline` で対象コミットを確認した上で該当範囲の差分を代替取得する
   - 今回 AIが生成した可能性のあるファイル
   - 以下が含まれていないか：
     * APIキー形式：sk-, pk-, AIza, ghp_, xoxb-, AKIA
     * パスワード、トークン、シークレット
     * データベース接続文字列（実際の認証情報含む）
     * プライベートなURL、IPアドレス
     * 実際のメールアドレス
     * JWTトークン（eyJで始まる）

2. .gitignore保護の確認
   - .env, .env.local, credentials.json などが除外されているか
   - 新規追加ファイルに機密情報ファイルがないか

3. ステージ済み・未コミットの変更が残っていないかの確認
   - `git status` で作業ツリーがクリーンであることを確認
   - 残っている場合はコミット漏れの可能性があるため、ユーザーに確認してから次に進む

4. 未pushコミット差分の確認
   - 対象コミット全体をスキャン
   - 正規表現で危険パターンを検出

## 除外（これらはOK）

- プレースホルダー：`YOUR_API_KEY`, `***`, `sk-...`
- example.com, test.com のメールアドレス
- 説明文中の用語（「APIキーを設定」など）

## 実行手順

1. `git status` でステージ済み・未コミットの変更が残っていないか確認
2. upstreamの設定有無を確認し、`git diff @{u}...HEAD`（未設定の場合は代替コマンド）で未pushコミットの差分を取得
3. 新規追加ファイルを重点的に精査
4. 検出した問題を具体的に報告（ファイル名、行番号、危険度）
5. 問題なしの場合も明示的に「安全」と報告

## 報告形式

問題検出時は：
- 📄 ファイル名
- Line X: 問題の種類
- 該当箇所（一部マスク）
- 危険度（高/中/低）

問題なしの場合：
「✅ セキュリティチェック完了。プッシュ安全。」

## プッシュ実行

チェック完了後、ユーザーの承認を得てからプッシュを実行します。
