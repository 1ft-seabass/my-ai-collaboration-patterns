## 新機能

### 1. 軽量セッション締め用アクション追加

**ファイル**: `docs/actions/doc_note_and_commit.md`

**概要**:
- ノート作成 + コミットを一気に実行する10段階フロー
- `@doc_note` → `@git_commit` の2回呼び出しが1回で完結
- 軽くキリッと終わるセッションに最適

**使用場面**:
- セッション終了時にノートとコミットだけしたい
- 申し送り作成は不要な軽量セッション
- 前後関係を明確にして効率的に作業完了

**従来の課題**:
- 2つのアクションを個別に呼ぶ手間
- 同じメッセージで呼ぶと前後関係が不明でバグりやすい

**解決策**:
- `doc_note.md` + `git_commit.md` の要素を統合
- 一気通貫のフロー

## 改善

### 2. セッション終了フローの文言改善

**ファイル**: `docs/actions/00_session_end.md`

**修正箇所**: 106行目（申し送り完了通知の次セッション開始メッセージ）

**Before**:
```
docs/letters/yyyy-mm-dd-hh-mm-ss-title.md を日本語で。
```

**After**:
```
docs/letters/yyyy-mm-dd-hh-mm-ss-title.md が申し送りファイルです。見てください。以降は日本語のセッションで進めます。
```

**改善点**:
- 「申し送りファイルです」と明示 → AIが翻訳と勘違いしない
- 「見てください」で読むべき対象と明確化
- 「以降は日本語のセッションで進めます」で言語設定も明確化

**効果**:
- AI協働におけるセッション開始時の文脈継承が確実に
- 申し送りを読まずに作業を進める問題を解消

### 3. アクション粒度設計の確立

**3段階の粒度**:

| アクション | 用途 | フロー | 重さ |
|-----------|------|------|------|
| `doc_note.md` / `git_commit.md` | 単機能 | ノート / コミット | 軽量 |
| `doc_note_and_commit.md` | 組み合わせ | ノート→コミット | 中量 |
| `00_session_end.md` | 完全フロー | ノート→コミット→申し送り | 重厚 |

**価値**:
- 様々な使用場面に対応
- ユーザーの手間削減
- バグ削減（前後関係が明確）
