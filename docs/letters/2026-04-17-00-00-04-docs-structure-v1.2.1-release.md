## セッション文脈サマリー

### 核心的な設計決定

- **for_branch_init.md の移動先**: `setup-pattern/docs-structure-for-branch/`
  - 理由: 0手順目の一回限りセットアップであり ongoing action ではない。廃止した `docs-structure-for-target-branch-only/` との違いは「複製でなく差分・特有動作のみ」という責務の明確化
  - 影響: `docs-structure/README.md` からリンクで参照する構造になった

- **元ディレクトリ（docs/notes/ 等）は一切触れない**
  - 理由: main で積んだコンテンツが含まれる可能性があり irreplaceable。触れる選択肢をAIに与えると事故が起きる
  - 設計: 削除・移動の選択肢を手順から排除。触れないことを明文化

- **TEMPLATE.md 取得は degit が正解（未実装）**
  - 現状: `cp` で元ディレクトリからコピー
  - 問題: 他人のプロジェクトでは TEMPLATE.md が存在しないか別物の可能性
  - 方針: `npx degit` で公式テンプレートを直接取得する方式に変更予定

### 次のセッションに引き継ぐべき「空気感」

- ゆるッとした会話ベースで設計議論を楽しむスタイル
- シンプルさ優先・同期コストを避ける設計哲学
- git が安全網なのでコード側は過剰な退避処理不要という合意
- `setup-pattern` はショートカット的なリンク集として発展させる構想あり（`docs-and-securecheck` など）
