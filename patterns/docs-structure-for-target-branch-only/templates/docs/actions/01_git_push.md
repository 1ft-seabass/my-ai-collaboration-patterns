このセッションでプッシュ前の最終チェックを実施します。

## チェック項目

1. ドキュメント類の機密情報チェック（最重要）
   - README.md, docs/letters, docs/notes 対象
   - 今回 AIが生成した可能性のあるファイル
   - 以下が含まれていないか：
     * APIキー形式：sk-, pk-, AIza, ghp_, xoxb-, AKIA
     * パスワード、トークン、シークレット
     * データベース接続文字列（実際の認証情報含む）
     * プライベートなURL、IPアドレス
     * 実際のメールアドレス
     * JWTトークン（eyJで始まる）

2. .gitignore保護の確認
   - .env, .env.local, credentials.json などが除外されているか
   - 新規追加ファイルに機密情報ファイルがないか

3. git diff確認
   - コミット内容全体をスキャン
   - 正規表現で危険パターンを検出

## 除外（これらはOK）

- プレースホルダー：`YOUR_API_KEY`, `***`, `sk-...`
- example.com, test.com のメールアドレス
- 説明文中の用語（「APIキーを設定」など）

## 実行手順

1. `git diff --cached` で変更内容を取得
2. 新規追加ファイルを重点的に精査
3. 検出した問題を具体的に報告（ファイル名、行番号、危険度）
4. 問題なしの場合も明示的に「安全」と報告

## 報告形式

問題検出時は：
- 📄 ファイル名
- Line X: 問題の種類
- 該当箇所（一部マスク）
- 危険度（高/中/低）

問題なしの場合：
「✅ セキュリティチェック完了。プッシュ安全。」

## プッシュ実行

チェック完了後、ユーザーの承認を得てからプッシュを実行します。
