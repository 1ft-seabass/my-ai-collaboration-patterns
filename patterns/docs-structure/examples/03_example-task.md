## 概要

本物の API 実装が完了したため、開発初期に作成した仮実装（モック・スタブ）ファイルを削除する。

## 背景・理由

- フロントエンド開発を先行させるため、最初は `server/mocks/users.js` でモックデータを使用していた
- Phase 1, 2 で本物の API 実装（`server/routes/users.js`）が完成
- モックファイルが残っていると、どちらが本物か混乱する
- コードベースをクリーンに保つため削除が必要

### 削除対象ファイル

```
server/
├── mocks/
│   ├── users.js          # ← 削除対象
│   └── products.js       # ← 削除対象（未使用）
└── __tests__/
    └── mocks/
        └── usersMock.js  # ← テストで使用中のため残す
```

## 確認項目・チェックリスト

### 削除前の確認

- [ ] `server/mocks/users.js` が現在使用されていないことを確認
  - `grep -r "mocks/users" server/` で検索
  - インポート文が残っていないか確認

- [ ] `server/mocks/products.js` が現在使用されていないことを確認
  - `grep -r "mocks/products" server/` で検索

- [ ] テストファイルで使用されているモックは残す
  - `server/__tests__/mocks/usersMock.js` はテストで使用中
  - このファイルは**削除しない**

### 削除後の確認

- [ ] アプリケーションが正常に起動するか確認
  - `npm run server` でバックエンドが起動
  - エラーが出ないか確認

- [ ] API が正常に動作するか確認
  - `curl http://localhost:3001/api/users` でデータ取得
  - 本物の API が応答するか確認

- [ ] テストが通るか確認
  - `npm test` でテストスイート実行
  - モックを使用しているテストが失敗していないか確認
