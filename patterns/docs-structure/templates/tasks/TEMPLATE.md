## 📖 テンプレート使用ガイド

### このディレクトリの役割
セッション跨ぎで継続するタスクや、長期的なTODOを管理します。

### 命名規則
```
docs/tasks/yyyy-mm-dd-hh-mm-ss-{title}.md
```

- **yyyy-mm-dd-hh-mm-ss**: 作成日時（例: 2026-02-01-14-30-00）
- **title**: タスクの内容を表すタイトル（ケバブケース推奨、30文字以内目安）
- 例: `2026-02-01-14-30-00-api-migration-task.md`

### FrontMatter の記入方法
```yaml
---
tags: [api, migration, backend]
status: open
priority: high
---
```

- **tags**: 3-5個推奨（技術キーワード、対象機能など）
- **status**: open / in-progress / completed / cancelled
- **priority**: high / medium / low

### いつ使うか
- セッションを跨ぐタスクがある
- 長期的なTODOを記録したい
- チェックリスト形式で進捗管理したい

### 検索方法
```bash
# タグで検索
grep -r "tags:.*api" docs/tasks/

# ステータスで検索
grep -r "status: open" docs/tasks/

# タイトルで検索
ls docs/tasks/ | grep migration
```

### 機密情報保護ルール
- API キー・パスワード・トークンは必ずプレースホルダー（`YOUR_API_KEY`等）で記載
- 実際の機密情報は絶対に含めない
- .env や設定ファイルの内容をそのまま転記しない
