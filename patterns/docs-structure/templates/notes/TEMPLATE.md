## 📖 テンプレート使用ガイド

### このディレクトリの役割
開発中の試行錯誤、技術調査、デバッグ記録など、**開発プロセスの詳細**を記録します。

### 命名規則
```
docs/notes/yyyy-mm-dd-hh-mm-ss-{title}.md
```

- **yyyy-mm-dd-hh-mm-ss**: 作成日時（例: 2026-01-31-14-25-30）
- **title**: 内容を表す簡潔なタイトル（ケバブケース推奨、30文字以内目安）
- 例: `2026-01-31-14-25-30-api-authentication-debug.md`

### FrontMatter の記入方法
```yaml
---
tags: [api, security, debug]
---
```

- **tags**: 3-5個推奨（技術キーワード、トピック、パターン名など）
- 既存ノートの tags を参考に統一感を保つ
- 将来的な検索性向上のため（250/1000ノート時代に有効）

### いつ使うか
- バグ修正の試行錯誤を記録したい
- 技術調査の結果をまとめたい
- 実装方法の選択理由を残したい
- 失敗した試みも含めて記録したい

### 検索方法
```bash
# タグで検索
grep -r "tags:.*api" docs/notes/

# タイトルで検索
ls docs/notes/ | grep authentication

# 内容で検索
grep -r "失敗した理由" docs/notes/
```

### 機密情報保護ルール
- API キー・パスワード・トークンは必ずプレースホルダー（`YOUR_API_KEY`等）で記載
- 実際の機密情報は絶対に含めない
- .env や設定ファイルの内容をそのまま転記しない
