ここまでの経緯をノートにまとめます。

## 📋 この指示書の理解チェック

作業を開始する前に、以下の内容を理解したことをチェックボックスで提示してください：

- [ ] ノート作成方針をユーザーに提案し、承認を得る
- [ ] docs/notes/TEMPLATE.md をベースにノートを作成する
- [ ] 機密情報は必ずプレースホルダーで記載する
- [ ] 小分けにしたほうが良い場面も検討する

理解できましたか？ゴーサインをください。

---

⚠️ 機密情報保護ルール

ノートに記載する情報について

- API キー・パスワード・トークンは必ずプレースホルダー(`YOUR_API_KEY`等)で記載
- 実際の機密情報は絶対に含めない
- .env や設定ファイルの内容をそのまま転記しない

## 実行手順

1. ノート作成方針の提案
   - ここまでの経緯をノートにまとめる方針を考え、ユーザーに提案
   - 全体の経緯だけでなく小分けにしたノートを作成したほうが良い場面もあるので、ユーザーに提案
   - セッションの流れでノートをすでに作成してるなど作る必要がないときも、ユーザーに提案
   - ユーザーの承認を得る

2. ノートの作成
   - docs/notes/README.md をみてノート作成ルールを把握
   - docs/notes/TEMPLATE.md をベースにノートを作成
   - 承認された方針に従って作成
   - 機密情報は必ずプレースホルダーで記載

3. ノート作成完了の報告
   - 作成したノートのファイルパスを報告
   - 簡潔な内容サマリーを提示
